# SSAI Visual Language Rules

Use this as visual context for image models, mockups, and section explorations. This is not product copy. It is the accent, material, and composition grammar SSAI should consistently use.

## Overall Feel

SSAI feels like mission control plus a competent teammate interface. It is calm, technical, inspectable, warm, and precise. It should look like software used around production infrastructure by serious engineers, not a marketing dashboard for AI automation.

The visual language is austere but not cold. It should have editorial confidence, quiet density, and small moments of sharp machine detail.

## Inspirations

- Datafruit-style thesis pages: warm editorial ground, large serif reading, scroll-highlight paragraphs, lots of breathing room.
- Pi.dev-style grid and technical austerity: thin grid, mono details, terminal-adjacent precision without becoming cyberpunk.
- Memory.store-style product storytelling: real tool ecosystem, restrained line illustrations, warm paper-like surface.
- Developer tools like Codex, Claude Code, Raycast, Linear, and GitHub: fast orientation, serious work objects, compact controls, readable state.
- Professional instruments and ledgers: ruled tables, audit records, timestamps, action trails, evidence panels.

Use these as tonal references, not as templates to clone.

## Color Rules

- Use warm off-white as the main ground: `#fffefc`.
- Use deep ink for primary text: `#111111` and `#222222`.
- Use muted gray text for secondary information: `#4d4d4d` and `#888888`.
- Use warm hairlines and borders: `#d8d4cc`.
- Use SSAI Blue `#254bf1` as the only strong accent. It is for active states, links, scan focus, selected rows, and primary CTAs. Keep it sparse.
- Use SSAI Coral `#FF3C5B` once per visible screen at most. It is a brand flourish, not a general accent color.
- Use status colors only as small dots, small pills, or tiny border accents. Never flood large areas with green, amber, or red.
- Meta Mode can use charcoal grounds: `#262626` and `#2f2f2f`, with warm off-white text.

## Color Anti-Rules

- No gradients.
- No purple-blue SaaS glow.
- No neon.
- No glassmorphism.
- No cold white page backgrounds.
- No large colored icon sets.
- No large green/red dashboard zones.
- No decorative color blobs, orbs, bokeh, or abstract AI halos.
- No colorful confetti-style integration logo clouds unless logos are deliberately muted or grayed.

## Typography Rules

- Use a Tiempos-like serif for major statements, hero lines, thesis paragraphs, section titles, and serious product moments.
- Use a Geist-like sans for body text, navigation, buttons, labels, and UI copy.
- Use a Geist Mono-like monospace for timestamps, run IDs, environment names, command snippets, channels, policies, evals, and machine labels.
- Typography should feel editorial plus technical: large serif statements paired with compact sans/mono interface detail.
- Use strong scale contrast in heroes and thesis sections. Use compact hierarchy inside product panels.
- Letter spacing should usually be normal. Uppercase labels can have slight tracking.

## Typography Anti-Rules

- No gradient text.
- No inflated generic SaaS H1 treatment.
- No playful rounded display fonts.
- No cyberpunk terminal-only typography.
- No giant monospace blocks as the main brand expression.
- No dense all-caps interfaces except for tiny metadata labels.

## Shape And Surface Rules

- Use thin ruled grids, hairline borders, tables, ledgers, panels, columns, and sidebars.
- Use rounded icons and restrained rounded rectangles. Corners should usually be 4px to 8px. Larger radii should be rare.
- Icons should be simple line icons or rounded mono-style symbols. Prefer single-color ink, muted gray, or SSAI Blue.
- Product surfaces should feel inspectable: records, threads, evidence, memory, policies, evals, simulation rows.
- Use card-white `#ffffff` only for lifted surfaces on top of the warm ground.
- Shadows should be minimal and structural. Hairlines and spacing should create most of the depth.
- If elevation is needed, use very soft small shadows, never glossy dramatic shadows.

## Shape And Surface Anti-Rules

- No prominent shadows.
- No heavy 3D depth.
- No stacked marketing cards.
- No nested cards inside cards.
- No pill-heavy interfaces.
- No overly rounded SaaS bubbles.
- No huge colorful icon tiles.
- No cartoon illustrations.
- No abstract robot or AI mascot.

## Layout Rules

- Prefer editorial sections with clear technical artifacts embedded inside them.
- Use a visible grid as an ambient field, especially in the hero, but keep it quiet.
- Use split surfaces when showing product context: thread on one side, evidence or operating state on the other.
- Use ledger/table layouts for economics, audit trails, memory, and comparisons.
- Use grayed-out competitor or category logos for quick market understanding. They should be muted and secondary, not a trust wall.
- The landing page should feel spacious above the fold but reveal technical specificity quickly.

## Layout Anti-Rules

- No generic centered SaaS hero with gradient blob.
- No dashboard wall full of charts.
- No marketing bento grid made of unrelated feature cards.
- No fake analytics charts just for decoration.
- No split stock-photo hero.
- No over-decorated background textures.

## Motion Rules

- Motion should suggest scanning, inspection, replay, memory formation, and system learning.
- Use slow grid drift, one crosshair at a time, a blue scan focus field, subtle row highlights, and scroll-linked thesis text.
- Motion should be purposeful and quiet. It should feel like instrumentation, not entertainment.
- Every animated composition should still look good as a static frame.

## Motion Anti-Rules

- No bounce.
- No elastic easing.
- No confetti.
- No particle storms.
- No aggressive parallax.
- No loading-spinner theater.
- No random floating UI cards.

## Icon Rules

- Use rounded, simple, single-color icons.
- Icons should feel like labels on an instrument panel, not app-store badges.
- Use line icons or compact geometric symbols.
- Keep icons small and paired with text when clarity matters.
- Integration logos can appear, but should be grayed, desaturated, or treated like reference marks unless the section specifically needs brand recognition.

## Icon Anti-Rules

- No multicolor decorative icon packs.
- No glossy icons.
- No emoji as product icons.
- No 3D icons.
- No mascot icons.
- No icon buttons that are visually louder than the work they control.

## Normal Mode

Normal Mode uses warm off-white, deep ink, sparse blue, hairlines, and calm product records. It shows customer infrastructure, action explanations, audits, deploys, incidents, cost hygiene, memory review, and rare overrides.

## Meta Mode

Meta Mode uses charcoal grounds and the same component grammar. It shows SSAI's own agents, runners, evals, simulations, memory quality, rejected commands, replay, policy drift, pause/resume, and continual learning. It should feel internal and focused, not like a separate cyberpunk product.

## One-Line Image Model Direction

Design SSAI as a warm, austere, editorial-technical interface: off-white paper ground, deep ink serif statements, Geist-like UI, sparse blue accents, thin ruled grids, rounded single-color icons, compact ledgers, inspectable product panels, no gradients, no prominent shadows, no colorful icon packs, no AI mascots, no generic SaaS dashboard.
